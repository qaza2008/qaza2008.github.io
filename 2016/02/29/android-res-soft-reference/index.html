android资源文件的软引用

发表于 2016-02-29   |   分类于 android   |  

概述

大家在项目中多少都会遇到这种情况:

  1. 同一个布局文件中会有一些相同的控件,但是这些控件的id不一致,如果要在项目中获取这些控件,多多少少得使用id硬代码来获取,如果这些控件非常多的话,估计这种使用id硬代码就有些不合理了.
  2. 同一个布局需要使用include引进同一个子布局多次,怎么实现.
首先解决第一个问题:
  1. 不要在layout中的控件使用android:id="@+id/xxx"这样的硬代码.
  2. 接下来需要在[res]-[values]-ids.xml 中定义<item name="xxx_1_1" type="id"></item>这样的id.
  3. 为layout的控件的id赋值android:id="@id/xxx_1_1,代码引用ids.xml中定义好的id.
  4. 以上是对一个id的引用策略,下面扩展到多个.

layout

          <include
              android:id="@id/xxxp1"
              layout="@layout/layout_xxxitem"

          <include
              android:id="@id/xxxp2"

          <include

              layout="@layout/layout_xxxitem"
              android:layout_height="match_parent"
              android:layout_width="0dp"
              android:layout_weight="1"></include>

          <include
              android:id="@id/xxxp4"
              layout="@layout/layout_xxxitem"
              android:layout_height="match_parent"
              android:layout_width="0dp"
              android:layout_weight="1"></include>

      </LinearLayout>

ids 


<item name="xxxp1" type="id"></item>
  <item name="xxxp2" type="id"></item>
  <item name="xxxp3" type="id"></item>
  <item name="xxxp4" type="id"></item>
  <string-array name="xxxp_group">
      <item>@id/xxxp1</item>
      <item>@id/xxxp2</item>
      <item>@id/xxxp3</item>
      <item>@id/xxxp4</item>
  </string-array>
